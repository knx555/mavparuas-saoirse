## 3. Chronological Naming Process

### 3.1 Project Concept (March 28, 2026, approx. 17:00 CET)

Markus described the project: a universal file organization and synchronization tool operating across all storage locations (Nextcloud/WebDAV, OneDrive, Dropbox, Hetzner Storage Box, USB, local SSDs, Raspberry Pi). The tool is designed as a **plugin/module** within the MAVPARUAS ecosystem. Strict privacy-by-design architecture, exclusively local LLMs, MIT license.

### 3.2 Naming Round 1 (March 28, 2026, approx. 17:15 CET)

Four name candidates were systematically evaluated:

| Candidate | Collision Risk | Reason for Exclusion |
| --- | --- | --- |
| **FRIEDA** | MEDIUM | Collision with "Frida" (frida.re), a well-known reverse-engineering toolkit in the security community |
| **ALMA** | HIGH | Collision with AlmaLinux, an enterprise Linux distribution (used by CERN, AWS, Red Hat ecosystem) |
| **DORA** | HIGH | Collision with EU Regulation 2022/2554 (Digital Operational Resilience Act, financial sector), Google DORA Metrics, Nickelodeon character |
| **SELMA** | LOW | Technically clean, but the developed acronym contained "Local Media Archives", which does not reflect the tool's universal scope |

**Decision**: All four candidates rejected — none met the criteria (collision-free AND conceptually fitting).

### 3.3 SANA Proposal by Markus (March 28, 2026, approx. 17:35 CET)

Markus proposed the name **SANA** (acronym: Selfsorting Analytic Normalizing Algorithm).

**Research result**: Direct collision with **NVIDIA NVlabs/Sana** — an open-source AI image/video generation framework with 5,000+ GitHub stars, presented as ICLR 2025/2026 Oral Paper. Additionally, **Sana Commerce** exists (listed in Gartner Magic Quadrant).

**Decision**: SANA rejected due to direct collision in the AI ecosystem.

### 3.4 Naming Round 2 — Irish Naming Tradition (March 28, 2026, approx. 17:45 CET)

Markus expressed the desire to follow the existing naming tradition in the MAVPARUAS project: the sister project is named **Aisling** (Irish: "dream/vision"). Irish names offer the advantage of being rare in the software ecosystem while carrying cultural depth.

Six candidates were systematically evaluated:

| **ROISIN** | "Little Rose" (Irish) | LOW | No relevant software collisions found |

| **AOIFE** | "Beautiful, Radiant" (Irish) | LOW | Clean, but not pursued further |

### 3.5 Finalist Selection by Markus (March 28, 2026, approx. 17:55 CET)

Markus selected two finalists for in-depth trademark research:

1. **SAOIRSE** — Favorite #1
2. **ROISIN** — Favorite #2

**Markus' rationale** (paraphrased):

5. **Acronym**: "Synchronizing All Origins with Intelligent Recognition, Sorting & Exchange" precisely describes the technical function and strengthens the descriptive character.
6. **Distinctiveness through context**: The name is always used in the context "MAVPARUAS SAOIRSE" or with descriptive text, never as a standalone brand.

**Effort**: Medium (approx. €300–800 for a professional trademark search with similarity analysis)
